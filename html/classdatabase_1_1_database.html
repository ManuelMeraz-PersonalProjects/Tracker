Detailed Description

The database singleton class is in charge all database queries.

Due to the way SQLite works, we want a single connection to the database up and running at all times. This class keeps the connection maintained.

auto database::Database::get_connection ( ) -> soci::session &
static

Returns a reference to the current database connection;.

Need to return a reference explicitly because this item is uncopyable. Attempts to copy the connection will try to call a private constructor and fail.

Usage:
auto& sql_connection = database::Database::get_connection();
sql_connection << "some sql command here";

Will throw a soci::sqlite3_soci_error if the command fails
